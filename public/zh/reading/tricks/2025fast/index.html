FAST

2026年3月4日 · 2 分钟 · map[name:Hongchen Ji] | 语言:

FAST：用于 Vision-Language-Action 模型的高效动作 Tokenization#

作者： Karl Pertsch, Kyle Stachowicz, Brian Ichter, Danny Driess, Suraj Nair, Quan Vuong, Oier Mees, Chelsea Finn, Sergey Levine
机构： Physical Intelligence, UC Berkeley, Stanford
年份： 2025

论文：
https://arxiv.org/pdf/2501.09747

1. 概览#

FAST 提出了一种新的 动作 tokenization 方法，用于 Vision-Language-Action（VLA）模型。

其出发点是当前最常见的动作离散化方法 —— binning discretization（分桶离散化） —— 存在明显问题。

传统方法通常：

  • 按 每个维度、每个时间步 对动作进行离散化

但这种方法在 高频机器人控制数据 上表现较差。

FAST 的核心思路是：

通过压缩动作信号，将冗余的连续控制信号转换为 更少但信息量更高的 token

FAST 使用了两个关键技术：

  • 离散余弦变换（DCT）
  • Byte Pair Encoding（BPE）

2. 背景与问题#

传统的 binning discretization 存在两个主要问题。

2.1 动作 token 高度相关#

机器人控制信号通常是连续且平滑的。

因此连续时间步的动作 token 高度相关。

这会导致：

  • 每个 token 的信息量很低
  • 模型容易直接 复制上一个 token

这样会削弱模型的学习信号，使训练目标变得低效。

2.2 token 序列过长#

在高频控制场景下：

  • 每个动作片段可能产生 数百个 token

这会带来多个问题：

  • 训练困难
  • 推理速度变慢
  • Transformer 上下文窗口被大量低信息 token 占用

FAST 的目标就是在 tokenization 之前 压缩动作序列。

3. 核心思想#

FAST 的核心思想是：

在频域对动作序列进行 tokenization。

具体流程是：

  1. 将动作序列通过 DCT 转换到频域
  2. 去除冗余的高频信息
  3. 使用 BPE 对频域表示进行压缩

最终得到：

更短、更高信息密度的 token 序列。

4. 方法#

4.1 动作归一化#

在 tokenization 之前，需要对动作进行归一化。

每个动作维度会根据数据集的：

  • 1% 分位数
  • 99% 分位数

映射到范围：

[ [-1,1] ]

这样可以避免极端值影响表示。

4.2 离散余弦变换（DCT）#

接下来，对每个动作维度分别应用 DCT。

DCT 将动作序列从：

时间域 → 频率域

一个动作轨迹可以表示为多个不同频率余弦函数的叠加。

关键观察：

  • 低频部分包含主要运动信息
  • 高频部分通常是冗余或噪声

因此可以通过忽略较小的高频系数来实现压缩。

4.3 构造矩阵表示#

对每个动作维度做 DCT 后：

  • 得到一个频率系数矩阵

然后：

  • 按 列优先（column-wise） 展平成一维序列

该序列即为压缩后的动作表示。

4.4 Byte Pair Encoding（BPE）#

接下来使用 BPE 对序列进一步压缩。

BPE 会学习：

  • 数据中频繁出现的系数组合

并将其合并为新的 token。

最终得到：

固定大小词表的动作 token。

需要注意：

  • FAST 中唯一需要学习的部分是 BPE 词表
  • 通常会 针对每个数据集单独训练

4.5 FAST+#

论文还提出了 FAST+。

FAST+ 是一个 通用机器人 tokenizer：

  • 在 100 万条多样化机器人动作数据 上训练
  • 已发布在 HuggingFace

FAST+ 不再依赖特定数据集，可以跨不同机器人系统使用。

5. 关键超参数#

FAST 有两个主要超参数。

5.1 DCT rounding scale#

控制 DCT 系数在压缩时的舍入强度。

5.2 BPE vocabulary size#

控制 token 序列压缩程度。

词表越大：

  • token 序列越短
  • 但训练复杂度也更高。

6. 实验结果#

论文报告了几个重要结论。

6.1 训练速度#

FAST 可以显著提升训练速度。

示例：

  • π0 vs π0-FAST
  • 训练速度提升 约 5 倍

原因是 FAST 生成的 token 序列 更短。

6.2 推理速度#

FAST 在推理阶段反而 更慢。

论文给出的例子：

  • π0：约 100 ms
  • π0-FAST：约 750 ms

原因包括：

  • 需要更多自回归 decoding
  • 使用大型语言模型 backbone
  • 而不是 π0 中较小的 action expert

6.3 token 压缩效果#

FAST 可以显著减少每个动作片段的 token 数量。

尤其是在 高频机器人数据 中。

此外：

BPE 可以避免大量 重复的 0 token，
这些 token 在传统 discretization 中经常出现，并会降低学习效率。

6.4 与其他 VLA 模型的兼容性#

FAST 不依赖特定模型结构。

它可以直接用于：

  • RT 系列
  • OpenVLA
  • π 系列

等多种 VLA backbone。

6.5 通用 tokenizer#

FAST+ 的实验结果表明：

通用机器人 tokenizer 的性能与数据集特定 tokenizer 相近。

这意味着未来可能存在：

统一的机器人动作 tokenizer。

7. 实际意义#

论文指出：

FAST 使得在 DROID 数据集上 可以训练出第一个能够 zero-shot 评估的通用机器人策略。

这说明：

动作 tokenization 的改进可以显著提升 VLA 模型的可扩展性。

8. 个人笔记#

核心 insight#

FAST 的核心洞察是：

机器人动作信号本身高度冗余。

因此直接在时间域进行离散化并不是最佳方法。

更好的方法是：

  1. 先转换到频域
  2. 压缩信号
  3. 再进行 tokenization
