RT-2：将互联网知识迁移到机器人控制的 Vision-Language-Action 模型#

作者：Anthony Brohan 等 机构：Google DeepMind 年份：2023

论文 / 博客 https://deepmind.google/discover/blog/rt-2-new-model-translates-vision-and-language-into-action/

Problem#

传统机器人策略通常只依赖 机器人交互数据（demonstrations 或 RL 数据）。

但与互联网规模的数据相比，机器人数据集通常非常小，因此这些策略往往只学到了：

  • 具体任务的操作模式
  • 局部的动作策略

而缺乏：

  • 语义知识
  • 常识推理
  • 对新任务的泛化能力

例如，一个机器人策略可能在训练过的任务上表现很好，但在以下情况中容易失败：

  • 遇到新的物体类别
  • 接收到新的语言指令
  • 需要理解物体用途（affordance）

换句话说：

传统机器人策略擅长学习 运动技能，但缺乏 世界知识（world knowledge）。

Why it matters#

大型视觉语言模型（VLM）已经在 互联网规模数据 上训练，并学习到了大量语义知识，例如：

  • 物体类别
  • 语言描述
  • 物体之间的语义关系
  • 常识推理模式

如果这些知识能够被迁移到机器人系统中，机器人可能可以：

  • 理解新的自然语言指令
  • 识别未见过的物体
  • 根据语义推理完成任务

然而传统机器人系统通常将以下模块 分开设计：

perception → language → control

这种结构使机器人策略无法直接利用 互联网规模的语义知识。

Attempt#

RT-2 提出了一个非常简单但影响深远的想法：

直接使用 Vision-Language Model 作为机器人策略模型。

作者没有设计新的机器人网络结构，而是让模型直接生成 机器人动作 token。

整体 pipeline：

image + language instruction ↓ Vision-Language Model ↓ action tokens

例如模型输出：

132 114 128 5 25 156

这些 token 表示离散化后的机器人控制指令，例如：

  • 末端执行器位置
  • 旋转角度
  • 夹爪开合

因此机器人控制问题被重新表述为：

在视觉和语言条件下的 token 预测问题。

Model#

Vision-Language Backbone#

RT-2 构建在大型视觉语言模型之上：

  • PaLM-E（12B 参数）
  • PaLI-X（5B / 55B 参数）

这些模型通常由两部分组成：

Vision Transformer（ViT）

Large Language Model（LLM）

组合后形成一个能够同时处理 图像 + 文本 的 Vision-Language Model。

Action Tokenization#

RT-2 不再使用单独的机器人策略网络，而是：

直接使用语言模型的输出头生成机器人动作 token。

动作被离散化为 token，并与文本 token 共享生成机制。

因此机器人控制被统一为：

条件序列生成问题（conditional sequence generation）。

Robot Dataset#

RT-2 使用的机器人训练数据来自 RT-1 数据集。

数据规模包括：

  • 13 台机器人
  • 约 130k demonstrations
  • 17 个月数据采集

需要注意：

RT-1 模型本身并没有被复用，RT-2 只使用了 RT-1 的数据。

Co-Fine-Tuning#

RT-2 训练中的关键技术之一是 co-fine-tuning（联合微调）。

模型在训练时同时使用两种数据：

  • 机器人交互数据
  • 原始视觉语言数据

原因是：

机器人数据规模较小，如果只用机器人数据训练，模型会：

  • 过拟合
  • 丢失原有的语义知识

论文中的 ablation 结果表明：

  • 只使用机器人数据训练 → 泛化性能明显下降
  • 使用 co-fine-tuning → 能够保留语义推理能力

Chain-of-Thought 推理#

RT-2 还继承了底层语言模型的 推理能力。

通过 chain-of-thought prompting，模型可以：

  1. 先对任务进行语义推理
  2. 推断物体属性
  3. 再生成最终动作

例如：

Instruction: “pick up the object that can hold liquid”

推理：

cup → 可以装液体

动作：

移动夹爪抓取杯子

说明语义推理可以影响机器人动作决策。

Failure / Limitation#

尽管 RT-2 展示了更好的泛化能力，但仍然存在一些限制。

运动技能仍依赖机器人数据#

互联网数据能够提供语义知识，但 无法直接提供新的操作技能。

机器人仍然需要：

  • demonstration 数据
  • 真实机器人交互

来学习：

  • manipulation dynamics
  • 接触控制

空间与物理理解仍然有限#

RT-2 在语义理解方面提升明显，但在以下方面仍然较弱：

  • 三维空间理解
  • 物理交互建模
  • 复杂 manipulation

动作离散化问题#

动作 tokenization 简化了建模，但也可能带来：

  • 控制精度降低
  • 离散化误差

Fix / Key Idea#

RT-2 的核心创新是：

将视觉-语言推理与机器人控制统一到同一个模型中。

从系统角度看，RT-2 相当于把两个系统合并：

PaLM-E → 语义推理

RT-1 → 机器人策略

最终形成一个 Vision-Language-Action（VLA）模型。

核心思想可以总结为：

机器人控制可以被视为多模态模型中的序列生成问题。

Results#

RT-2 在 未见任务泛化能力 上取得了显著提升。

主要结果包括：

  • 平均泛化性能提升约 2×
  • 在完全 OOD 任务 上 PaLI-X 版本的 RT-2 比最佳 baseline 提升约 3×

提升能力包括：

  • 识别未见过的物体
  • 理解自然语言指令
  • 利用互联网语义知识完成任务

Takeaway#

RT-2 展示了一个非常重要的方向：

机器人策略可以被建模为 Vision-Language 模型中的 token 预测问题。

通过这种方式，机器人可以直接利用互联网规模模型中的语义知识。

这一思想也影响了后续许多 VLA 模型，例如：

  • Octo
  • OpenVLA
  • π0
