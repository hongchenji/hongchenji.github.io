Diffusion Policy and Flow Matching: Foundations

Diffusion Policy and Flow Matching: Foundations#

This note summarizes the theoretical foundations behind diffusion models, diffusion policy, and flow matching, which are widely used in modern generative models and robot policy learning.

The goal is to understand how generative models can learn complex distributions by transforming noise into structured data.

1. Background Concepts#

Before understanding diffusion models and flow matching, several fundamental concepts are required.

KL Divergence#

KL divergence measures the difference between two probability distributions.

[ KL(q(x) || p(x)) = \int q(x) \log \frac{q(x)}{p(x)} dx ]

In machine learning it is often written as:

[ KL(q||p) = \mathbb{E}_{q(x)}[\log q(x) - \log p(x)] ]

Key intuition:

  • KL divergence measures how well distribution p approximates q
  • KL is always non-negative
  • KL = 0 only when the two distributions are identical

Maximum Likelihood#

Maximum likelihood estimation (MLE) learns model parameters by maximizing:

[ \log p_\theta(x) ]

This corresponds to minimizing the negative log likelihood.

In practice, many generative models approximate this objective using variational inference.

Variational Inference and ELBO#

Directly computing:

[ p*\theta(x) = \int p*\theta(x|z)p(z)dz ]

is usually intractable.

Therefore we introduce an approximate posterior:

[ q_\phi(z|x) ]

The log likelihood can be decomposed into:

[ \log p*\theta(x) = ELBO + KL(q*\phi(z|x) || p_\theta(z|x)) ]

Because KL divergence is non-negative:

[ \log p_\theta(x) \ge ELBO ]

Thus training becomes maximizing ELBO.

The ELBO is:

[ ELBO = \mathbb{E}{q\phi(z|x)}[\log p_\theta(x|z)]

  • KL(q_\phi(z|x) || p(z)) ]

This contains two components:

  1. Reconstruction loss

[ \mathbb{E}{q\phi(z|x)}[\log p_\theta(x|z)] ]

  1. KL regularization

[ KL(q_\phi(z|x) || p(z)) ]

Reparameterization Trick#

To sample from

[ z \sim \mathcal{N}(\mu,\sigma^2) ]

we rewrite it as:

[ z = \mu + \sigma \epsilon, \quad \epsilon \sim \mathcal{N}(0,1) ]

This allows gradients to flow through stochastic sampling.

2. Diffusion Models#

Diffusion models generate data by gradually removing noise from a Gaussian distribution.

They consist of two processes:

  • forward diffusion process
  • reverse denoising process

3. Forward Diffusion Process#

The forward process gradually adds Gaussian noise to data.

At timestep (t):

[ x_t = \sqrt{\bar{\alpha}_t} x_0 + \sqrt{1-\bar{\alpha}_t}\epsilon ]

where

[ \epsilon \sim \mathcal{N}(0,I) ]

and

[ \bar{\alpha}t = \prod{s=1}^t \alpha_s ]

As (t) increases:

[ \bar{\alpha}_t \rightarrow 0 ]

Thus

[ x_T \sim \mathcal{N}(0,I) ]

Regardless of the original image.

This works because repeated addition of independent noise drives the distribution toward a Gaussian (related to the central limit theorem).

4. Reverse Diffusion Process#

The reverse process learns to remove noise step by step.

We train a neural network:

[ \epsilon_\theta(x_t, t) ]

to predict the noise added at each timestep.

Training objective:

[ L = \mathbb{E}{x_0,\epsilon,t} \left[ ||\epsilon - \epsilon\theta(x_t,t)||^2 \right] ]

This is simply MSE between true noise and predicted noise.

During generation:

  1. sample noise

[ x_T \sim \mathcal{N}(0,I) ]

  1. iteratively denoise

[ xT \rightarrow x{T-1} \rightarrow … \rightarrow x_0 ]

5. Diffusion Model Architecture#

Typical diffusion models consist of three components.

Text Encoder#

Maps text prompts to embeddings.

Common choices:

  • CLIP
  • BERT
  • GPT
  • T5

Text encoder capacity strongly influences generation quality.

Generation Model#

The core diffusion network predicts noise.

Typical architecture:

  • U-Net

Inputs include:

  • noisy image
  • timestep
  • text embedding

Output:

  • predicted noise

Decoder#

If diffusion is performed in latent space (e.g., Stable Diffusion), a decoder converts latent representations back into images.

Example pipeline: image → encoder → latent latent → diffusion latent → decoder → image

6. Evaluation Metrics#

Two common metrics are used to evaluate generative models.

FID Score#

Fréchet Inception Distance measures similarity between real and generated images.

Procedure:

  1. generate many images
  2. extract features using a CNN
  3. approximate both distributions as Gaussians
  4. compute Fréchet distance

Lower FID indicates better generation quality.

CLIP Score#

Measures alignment between generated images and text prompts.

CLIP encodes both text and image into embeddings.

Similarity between embeddings indicates semantic alignment.

7. Diffusion Policy in Robotics#

Diffusion models can also be used for robot action generation.

Instead of generating images, the model generates action trajectories.

Forward process:

  • add Gaussian noise to action sequences

Reverse process:

  • learn to remove noise and reconstruct actions

This produces smooth multimodal policies, which are useful for complex robot behaviors.

8. Flow Matching#

Flow Matching is an alternative formulation of diffusion-based generative modeling.

Instead of learning discrete denoising steps, flow matching learns a continuous vector field that transforms noise into data.

Core Idea#

Let:

[ x0 \sim p{data} ]

and

[ x_1 \sim \mathcal{N}(0,I) ]

Flow matching defines a continuous path:

[ x_t = (1-t)x_0 + tx_1 ]

The model learns a velocity field:

[ v_\theta(x_t,t) ]

such that:

[ \frac{dxt}{dt} = v\theta(x_t,t) ]

Training objective:

[ \mathbb{E} || v_\theta(x_t,t) - v^*(x_t,t) ||^2 ]

where (v^*) is the true velocity field.

Advantages of Flow Matching#

Compared with diffusion models:

  1. fewer sampling steps
  2. continuous-time formulation
  3. easier training objective
  4. faster generation

Because of these properties, flow matching has recently become popular in:

  • image generation
  • video generation
  • robot control policies

9. Personal Understanding#

Diffusion and flow matching both solve the same problem:

learning a transformation from simple noise distributions to complex data distributions.

Diffusion: noise → many denoising steps → data

Flow Matching: noise → continuous flow → data

In robotics, these models allow policies to generate smooth action trajectories, which is particularly useful for manipulation and control tasks.
