FAST: Efficient Action Tokenization for Vision-Language-Action Models#

Authors: Karl Pertsch, Kyle Stachowicz, Brian Ichter, Danny Driess, Suraj Nair, Quan Vuong, Oier Mees, Chelsea Finn, Sergey Levine
Institutions: Physical Intelligence, UC Berkeley, Stanford
Year: 2025

Paper:
https://arxiv.org/pdf/2501.09747

1. Overview#

FAST proposes a new action tokenization scheme for Vision-Language-Action (VLA) models.

The motivation comes from the limitations of the most common action tokenization method: binning discretization, where actions are discretized per dimension and per timestep.

This approach works poorly for high-frequency robot data, which leads to inefficient training and poor learning signals.

FAST addresses this by introducing a compression-based tokenization method that transforms action trajectories into compact token sequences using:

  • Discrete Cosine Transform (DCT)
  • Byte Pair Encoding (BPE)

The goal is to compress redundant action signals into fewer, high-information tokens.

2. Motivation#

Standard binning discretization suffers from several problems.

1. Highly correlated action tokens#

Robot control signals are typically smooth and continuous.

This means that consecutive action tokens are strongly correlated, which leads to:

  • low marginal information per token
  • models simply copying the previous token

As a result, the prediction objective becomes weak and learning becomes inefficient.

2. Very long token sequences#

High-frequency control produces hundreds of tokens per action chunk, which causes:

  • difficult training
  • slow inference
  • inefficient use of the transformer context window

FAST attempts to solve these problems by compressing the action trajectory before tokenization.

3. Core Idea#

The core idea of FAST is to perform frequency-space action sequence tokenization.

Instead of tokenizing actions directly in the time domain, FAST:

  1. converts action trajectories into frequency space using DCT
  2. removes redundant high-frequency information
  3. compresses the resulting representation using Byte Pair Encoding

This produces a much shorter sequence of high-information tokens.

4. Method#

4.1 Action Normalization#

Before tokenization, actions are normalized.

Each action dimension is mapped to the range:

[ [-1, 1] ]

using the 1st and 99th percentile values of the dataset.

This prevents extreme values from dominating the representation.

4.2 Discrete Cosine Transform (DCT)#

Next, FAST applies DCT to each action dimension independently.

DCT converts the action trajectory from the time domain to the frequency domain.

The trajectory is represented as a sum of cosine functions with different frequencies.

Key idea:

  • low-frequency components capture most of the motion structure
  • high-frequency components often contain redundant or noisy information

Therefore, the representation can be compressed by removing small frequency components.

4.3 Matrix Construction#

After applying DCT to each action dimension:

  • the frequency coefficients form a matrix
  • this matrix is flattened column-wise into a 1D sequence

This sequence represents the compressed action trajectory.

4.4 Byte Pair Encoding (BPE)#

The flattened sequence is then further compressed using Byte Pair Encoding (BPE).

BPE learns frequently occurring coefficient patterns and merges them into tokens.

The result is a fixed-size vocabulary of action tokens.

Important properties:

  • the only learned component of FAST is the BPE vocabulary
  • the vocabulary is typically trained per dataset

4.5 FAST+#

The paper also introduces FAST+, a universal robot tokenizer.

FAST+ is trained on 1 million diverse robot actions and released on HuggingFace.

Unlike dataset-specific FAST tokenizers, FAST+ can be applied across different datasets and robot platforms.

5. Key Hyperparameters#

FAST has two main hyperparameters:

1. DCT rounding scale#

Controls how aggressively frequency coefficients are rounded during compression.

2. BPE vocabulary size#

Determines how strongly the token sequence is compressed.

6. Results#

The paper reports several important findings.

Training speed#

FAST significantly reduces training time.

Example result:

  • 5× faster training
  • comparing π0 vs π0-FAST

The main reason is that FAST produces much shorter token sequences.

Inference speed#

Inference becomes slower with FAST.

Example numbers reported:

  • π0: 100 ms
  • π0-FAST: 750 ms

This is because FAST requires:

  • more autoregressive decoding steps
  • the use of a large language backbone instead of the smaller action expert used by π0

Token compression#

FAST significantly reduces the number of tokens per action chunk, especially for high-frequency robot data.

Another benefit is that BPE avoids long runs of repeated zero tokens, which are common in naive discretization methods and slow down learning.

Compatibility with other VLA models#

FAST is not tied to a specific architecture.

The tokenizer can be applied to other VLA backbones with minimal changes.

Universal tokenizer performance#

FAST+ performs similarly to dataset-specific tokenizers.

This suggests that a universal robot tokenizer may be feasible.

7. Practical Impact#

One notable result mentioned in the paper is that FAST enables the first generalist policy on the DROID dataset that can be evaluated zero-shot.

This suggests that improved action tokenization can significantly improve scalability of VLA training.

8. Personal Notes#

Key insight#

The main insight of FAST is that robot action signals are highly redundant.

Instead of discretizing them directly in the time domain, it is more efficient to:

  1. transform them into frequency space
  2. compress them
  3. tokenize the compressed representation
