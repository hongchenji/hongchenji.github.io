RT2

March 4, 2026 · 4 min · map[name:Hongchen Ji]

RT-2

by Google DeepMind

The RT-2 paper introduces a novel vision-language-action (VLA) model called Robotic Transformer 2 (RT-2) that learns from both web and robotics data. The key idea, which is simple, is captured by this animation. A VLM (e.g., PaLI-X or PaLM-E) is trained on internet text, internet images (e.g., VQA), and then finetuned on robot data to output actions literally as text tokens (“132 114 128 5 25 156”) to create a “vision-language-action” model. The RT-1 in the animation is confusing a bit, but it is just implying that some of the robot data was used from RT-1, but nothing from the RT-1 model was used.

Architecture

Action format

The architecture of RT-2 consists of the following components:

  • Vision-Language Model (VLM): RT-2 uses a VLM as its backbone, which enables it to plan from both image and text commands, enabling visually grounded planning (Palm-E and PaLI-X) ( The Vit and LLM combined are just a VLM)
    • PaLM-E 12B
    • PaLI-X 5B, 55B
  • Robotic Data: RT-2 incorporates robotic data to improve its understanding of the physical world and its ability to output actions literally as text tokens from images directly
    • RT-1 data
    • Human demos
    • 13 robots
    • 17 month
  • Chain-of-Thought Reasoning: RT-2 incorporates chain-of-thought reasoning, allowing it to perform multi-stage reasoning and problem-solving tasks
  • Co-fine tuning: To make sure that RT-2 doesn’t forget various concepts baked into the VLM weights, some of the VLM data are kept around during the robot action fine-tuning phase (co-fine-tuning)

RT2w/PaLI-X-55B ablations

Training on robot data only, or even fine tuning on robot data, is not as good as co-fine tuning. If you only train on robot data, it will not work, because it will over-fit, since the data is small

Results

~2x better avg generalization than the next best baseline.

Tasks that are completely out of distribution of the robot data collected. RT-2 PaLI-X was 3x better than the next best baseline.

Chain-of-thought to get the model to “think through” the task and then output the actions

Interesting thoughts & conclusion from RT-2

  1. Instead of designing custom model architectures for robotics the solution was no architecture -> just use a VLM without adding any new layers.
  2. The best VLMs might be the most generalizable robotic controllers.
  3. With a little bit of robot data, it is possible to transfer semantic concepts from vision language web-scale data to robot actions.
  4. While RT-2 shows the transfer of semantic and visual concepts, it does not acquire the ability to perform new motions, indicating that further work is needed in this area.
  5. RT-2 is essentially functioning as an end-to-end version of what PaLM-E + RT-1 accomplish in one model.
