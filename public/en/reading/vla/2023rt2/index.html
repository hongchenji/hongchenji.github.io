RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control#

Authors: Anthony Brohan et al.

Organization: Google DeepMind Year: 2023

Paper / Blog https://deepmind.google/discover/blog/rt-2-new-model-translates-vision-and-language-into-action/

Problem#

Robot policies trained purely on robot datasets often struggle to generalize to new tasks.

Most robot learning systems rely on:

  • imitation learning
  • reinforcement learning
  • robot demonstration datasets

However, robot datasets are typically orders of magnitude smaller than the web-scale datasets used to train modern vision and language models.

As a result, robot policies often lack:

  • semantic understanding
  • language grounding
  • commonsense reasoning

For example, a robot trained on demonstrations might fail when asked to:

  • pick up an unfamiliar object
  • interpret a new instruction
  • reason about object affordances

The core limitation is that robot policies typically learn motor behavior, but not semantic knowledge about the world.

Why it matters#

Large vision-language models (VLMs) trained on internet-scale data already contain rich knowledge about:

  • objects and categories
  • language instructions
  • semantic relationships
  • everyday reasoning

If this knowledge could be transferred into robot control systems, robots could potentially:

  • generalize to unseen objects
  • interpret flexible natural-language instructions
  • reason about tasks instead of memorizing them

Traditional robotics pipelines separate:

perception → language understanding → control

This separation prevents robot policies from directly benefiting from the semantic knowledge learned by large multimodal models.

Attempt#

RT-2 proposes a surprisingly simple idea:

Use a large Vision-Language Model (VLM) as the robot policy itself.

Instead of designing a custom robotics architecture, the model generates robot actions as tokens, similar to how language models generate text.

Pipeline:

image + language instruction ↓ Vision-Language Model ↓ action tokens

Example output tokens:

132 114 128 5 25 156

These tokens represent discretized robot commands such as:

  • end-effector translation
  • rotation
  • gripper state

Robot control is therefore reframed as a token prediction problem.

Model#

Vision-Language Backbone#

RT-2 builds on large VLMs including:

  • PaLM-E (12B parameters)
  • PaLI-X (5B / 55B parameters)

These architectures combine:

Vision Transformer (ViT)

  • Large Language Model (LLM)

forming a multimodal model capable of reasoning jointly over images and text.

Action Tokenization#

Robot actions are discretized into tokens.

Each action token corresponds to a component of the robot command, including:

  • end-effector translation
  • orientation
  • gripper open / close

Instead of using a separate policy head, RT-2 uses the language model output head to generate these tokens.

This allows robot control to be treated as:

sequence generation conditioned on visual observations and language instructions.

Robot Dataset#

RT-2 is trained using robot interaction data from the RT-1 dataset.

Dataset statistics:

  • 13 robots
  • ~130k demonstrations
  • 17 months of data collection

Important detail:

The RT-1 model is not reused, only the dataset.

Co-Fine-Tuning#

A key training technique used in RT-2 is co-fine-tuning.

The model is trained simultaneously on:

  • robot interaction data
  • original vision-language datasets

This is necessary because robot datasets alone are relatively small.

Training only on robot data leads to:

  • overfitting
  • loss of semantic knowledge

Experiments show that:

  • robot-only training performs significantly worse
  • co-fine-tuning preserves semantic reasoning ability

Chain-of-Thought Reasoning#

RT-2 also benefits from the reasoning capabilities of large language models.

With chain-of-thought prompting, the model can:

  1. reason about the task
  2. infer object properties
  3. generate the correct robot action tokens

Example reasoning pattern:

Instruction: “pick up the object that can hold liquid”

Reasoning:

cup → holds liquid

Action:

move gripper toward cup

This demonstrates how semantic reasoning can influence robot actions.

Failure / Limitation#

Despite its improvements, RT-2 still has several limitations.

Motion skills still require robot data#

Internet-scale data provides semantic knowledge but does not automatically teach robots new manipulation skills.

Motor control still depends heavily on:

  • robot demonstrations
  • physical interaction data

Limited spatial and physical reasoning#

RT-2 improves semantic understanding but still struggles with:

  • 3D spatial reasoning
  • physical interaction modeling
  • complex manipulation dynamics

Discretized control#

Action tokenization simplifies modeling but introduces potential drawbacks:

  • limited control precision
  • discretization artifacts

Fix / Key Idea#

The key conceptual innovation of RT-2 is unifying vision-language reasoning and robot control inside a single model.

Conceptually, RT-2 merges two earlier systems:

PaLM-E → semantic reasoning

RT-1 → robot policy learning

into a single Vision-Language-Action (VLA) model.

The main insight is:

robot control can be treated as sequence generation in a multimodal model.

This allows robots to leverage the semantic knowledge stored in internet-scale vision-language models.

Results#

RT-2 demonstrates strong improvements in generalization to unseen tasks.

Key findings include:

  • ~2× improvement in average generalization performance
  • On tasks outside the training distribution, the PaLI-X version achieves ~3× better performance than the next best baseline

Observed improvements include:

  • reasoning about unfamiliar objects
  • interpreting natural language instructions
  • transferring semantic knowledge learned from web data

Takeaway#

RT-2 shows that robot policies can be formulated as:

token prediction problems using large vision-language models.

Instead of building specialized robot architectures, the system reuses large multimodal models and trains them to output robot actions.

This idea directly influenced later Vision-Language-Action (VLA) models, including:

  • Octo
  • OpenVLA
  • π0
