π0

by Physical Intelligence

The authors aim to train a single generalist robot policy (foundation model) that can perform a wide range of different skills and control a wide range of different robots. The model can either be used out-of-the-box or finetuned to new downstream tasks. They propose to use a novel flow-matching architecture (action expert) on top of a VLM pretrained on internetscale semantic knowledge.

Architecture & Method

  • they take inspiration from Transfusion and separate weights in two sets:
    • weights for image and text (pretrained VLM)
    • robot-specific weights, thus actions & states, (action expert)
  • they employ the biggest to-date cross-embodiment dataset composed from curated OXE data, DROID, Bridge and a big own dataset with 7 embodiments and 68 tasks
    • as there are different embodiments, the configuration vector and action vector always have size of biggest c-space, robots with less DoF receive zero padding
  • during pre-training of the base model language labels get used: task name + segment annotation (=fine-tuned labels for sub-trajectories, around 2sec in length)

Results

  • learning of very long tasks (up to 10 min possible)
  • unclear how much and what kind of training data is used to work on all tasks reliably as well as how much universality is achievable
  • evaluation uses partial scores of success between 0 and 1

Network#

pi0#

  • Network
    • PaliGemma
      • based on Gemma
        • multi query attention
        • width 2048
        • depth 18
        • mlp dim 16384
        • num heads 18
        • num kv heads 1
        • head dim 256
      • used in pi0
        • width 1024
        • mlp dim 4096
        • 3b parameters
  • additional input and output projections for the robotics-specific tokens,
    • state vector qt
    • action vectors At = [at, …, at+H−1] with noise
      • H = 50 as action horizon
  • an additional MLP for incorporating the flow matching timestep information τ
    • W3⋅swish(W2⋅concat(W1⋅a_t′τ,φ(τ)))
    • action vector mapped to embedding dimension
  • blockwise causal attention mask
    • flow
      • first Il
      • then qt
      • last At
    • full bidirectional attention inside block and previous block
    • cannot attend to tokens in future blocks

pi0.5#

  • Network
    • π₀.₅ uses a separate MLP for projecting τ only and then applies adaptive RMSNorm to inject the timestep information to each layer of the action expert.
    • FAST action tokens attend to this prefix and auto regression to previous action
    • action expert embedding
      • attend to prefix and one another
      • no attend to FAST action token

pi0.5#

  • FAST tokenizer
    • use action tokenization accelerate training
    • main idea
      • compress action signals before training
      • DCT discrete cosine transform encoding
        • time to frequency area
        • low frequency pass
      • avoid simple action repeating
    • mechanism
      • first normalize 1 to 99 quantile of action to -1 to 1
      • then apply DCT on each action dimension seperately
      • then flatten matrix
      • then train Byte Pair Encoder to compress the flattend matrix to fixed size output vocabulary
  • Loss
    • for training use discrete action
      • cross entropy
      • autoregressive sampling of tokens
      • FAST tokenizer
        • attention matrices ensure actions not attend each other
    • for inference use continuous action
      • iterative integration of flow field during inference
      • flow-matching
    • minimize combined loss
      • H cross entropy between text token and predict logits
        • FAST action token included
      • F flow matching loss from smaller action expert
      • alpha trade-off parameter
  • Training
    • pretraining
      • alpha = 0
      • model is a standard VLM
      • predict next token of
        • text
        • object locations
        • FAST encoded action tokens
    • posttraining
      • add additional expert weights alpha 10
      • flow matching
    • inference
      • standard autoregressive decoding for l^
      • then 10 conditioned denoising step for a
      • zero padding for robot with less joints

💡 2. 方法核心 / Core Method#

  • 作者提出的思路 / Main idea proposed by the authors:

    -

    - π0 model consists of

        - a language model transformer backbone

            - standard late fusion VLM recipe

            - image encoders embed image observations into the same embedding space as language tokens.

    - robotics-specific inputs and outputs

        - proprioceptive state and robot actions.

    - conditional flow matching to model the continuous distribution of actions.
