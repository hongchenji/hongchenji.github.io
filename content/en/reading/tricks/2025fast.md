# FAST: Efficient Action Tokenization for Vision-Language-Action Models

**Authors:** Karl Pertsch et al.
**Institutions:** Physical Intelligence, UC Berkeley, Stanford
**Year:** 2025

Paper: [https://arxiv.org/pdf/2501.09747](https://arxiv.org/pdf/2501.09747)

---

# Problem

Vision-Language-Action (VLA) models often represent robot actions using **per-dimension binning discretization**.

In this approach:

- each action dimension is discretized independently
- each timestep becomes a token

While simple, this method becomes problematic for **high-frequency robot control data**.

Typical issues include:

- very long token sequences
- highly redundant tokens
- inefficient transformer training

Robot control signals are usually **smooth trajectories**, meaning consecutive action values are strongly correlated.

As a result, token sequences contain very little new information per timestep.

---

# Why it matters

In VLA models, the **action token sequence length directly affects training efficiency**.

If actions are tokenized naïvely:

- transformers must process extremely long sequences
- context windows are wasted on redundant signals
- learning signals become weak because tokens are easy to predict

This slows down both:

- training
- scaling of robot foundation models

Therefore a key question is:

> Can robot actions be tokenized in a way that preserves information while reducing sequence length?

---

# Attempt

FAST proposes a **compression-based action tokenization pipeline**.

Instead of tokenizing actions directly in the time domain, FAST first transforms and compresses the action trajectory.

The pipeline consists of three main steps:

1. normalize action trajectories
2. transform them into frequency space
3. compress the representation into discrete tokens

This produces a much shorter sequence of **high-information tokens**.

---

# Method

## Action Normalization

Each action dimension is normalized to:

[-1,1]

using the **1st and 99th percentile** values from the dataset.

This avoids extreme outliers dominating the representation.

---

## Discrete Cosine Transform (DCT)

FAST applies **DCT** to each action dimension independently.

DCT converts the trajectory from:

- time domain

into:

- frequency domain

A trajectory can be expressed as a weighted sum of cosine functions with different frequencies.

Key observation:

- most meaningful motion is captured by **low-frequency components**
- high-frequency components often contain noise or redundant detail

This allows the representation to be compressed.

---

## Matrix Construction

After DCT is applied to all action dimensions:

- the frequency coefficients form a matrix

This matrix is then **flattened column-wise** into a sequence.

The flattened sequence represents the compressed action trajectory.

---

## Byte Pair Encoding (BPE)

The flattened representation is further compressed using **Byte Pair Encoding (BPE)**.

BPE learns frequently occurring patterns and merges them into tokens.

The result is a vocabulary of action tokens.

Important properties:

- the only learned component is the **BPE vocabulary**
- tokenizers are typically trained **per dataset**

---

# FAST+

The paper also introduces **FAST+**, a universal robot tokenizer.

FAST+ is trained on roughly **1 million robot action trajectories**.

Unlike dataset-specific tokenizers, FAST+ can be applied across:

- different datasets
- different robot platforms

This suggests that a **universal robot action tokenizer** may be possible.

---

# Results

## Training speed

FAST significantly reduces training time.

Example comparison:

- π0 baseline
- π0 with FAST tokenization

Reported improvement:

~5× faster training

The primary reason is the reduction in token sequence length.

---

## Token compression

FAST produces **much shorter action token sequences** compared to binning.

It also avoids long runs of repeated tokens, which commonly appear when discretizing smooth signals.

---

## Inference cost

Interestingly, inference becomes slower in some settings.

Example numbers reported in the paper:

- π0: ~100 ms
- π0-FAST: ~750 ms

This is partly due to:

- more autoregressive decoding
- reliance on the language backbone instead of a lightweight action head

---

# Failure / Limitation

Despite its benefits, FAST also introduces tradeoffs.

### Higher inference latency

Compression improves training efficiency but may increase decoding cost.

### Dataset dependence

FAST tokenizers are often trained for a specific dataset.

### Additional preprocessing

The pipeline introduces extra preprocessing steps such as DCT and BPE training.

---

# Key Idea

FAST reframes action tokenization as a **signal compression problem** rather than simple discretization.

Instead of tokenizing raw trajectories, the system first compresses the signal in frequency space and then tokenizes the compressed representation.

---

# Takeaway

The core insight of FAST is:

robot actions are highly redundant signals.

Efficient VLA training therefore benefits from:

trajectory → frequency transform → compression → tokenization

By reducing sequence length while preserving structure, FAST improves the scalability of Vision-Language-Action models.
