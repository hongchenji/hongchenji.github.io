**Embodied Chain-of-Thought (ECoT)**

_by UC Berkeley, Stanford, University of Warsaw_

Other works usually output raw low-level controls, however to solve unfamiliar and complex tasks the robot should not just use such a “muscle memory” but reason instead. VLMs are proficient in iteratively solving problems (especially using chain-of-thought [CoT] prompting) but transferring CoT naivly to VLAs does not perform well as training samples are too simple and purely semantic in reasoning. ECoT includes visual grounding for plans using the environment and robot state (bounding boxes and EE position) together with CoT to make the robot think step-by-step about the actions instead of just giving an answer.

**Architecture / Method**

- combines standard linguistic COT (to break task instruction into sub-tasks) with embodied CoT (steps that require grounding the policy’s reasoning in robot state and scene)
- proposes pipeline for generating synthetic training data:
  - concatenate 1) original instruction with 2) scene description (Prismatic-7B VLM) and pass it to 3) Grounding DINO to detect objects + bounding boxes
  - 4. generate low-level action primitives (next 4 time steps) using robot propriception and 5) detect 2D EE position (OWLv2, SAM, RANSAC)
  - Gemini 1.0 prompted for final reasoning chain
  - pipeline was run for 7 days on Bridge v2 dataset (60k demonstrations)
- model needs to predict additional tokens for reasoning, thus has lower control frequency → keep parts of reasoning chain (e.g. high-level plan) fixed for multiple steps
  - synchronous: predict high-level every N steps
  - asynchronous (preferred): have two policies where first one continuously updates high-level policy while second one uses most recent reasoning chain and predicts low-level reasoning steps and actions
- provides increased interpretability for users by revealing faulty reasoning
  - user is allowed to interrupt policy execution once and give natural language feedback → re-prompt via ChatGPT to produce correct reasoning chain

**Results**

- 28% increased success rate over OpenVLA without additional training data
- even though ECoT reasoning is only trained on one embodiment, reasoning works on other robots (probably due to VLM pretraining)
