**Moo: Open-World Object Manipulation using Pre-Trained Vision-Language Models**

_by Google DeepMind_

MOO" (Manipulation of Open-World Objects) improves upon "RT-1" by leveraging the rich latent spaces of Vision-Language Models (VLMs) to manipulate open-world objects. Unlike RT-1, MOO simplifies generalization by using a segmentation mask representation that is identical for both previously seen and novel unseen objects, thereby facilitating generalization. MOO exhibits generalization to unseen objects at a level comparable to that for seen objects, representing a significant improvement over RT-1.

**Architecture**

The architecture of MOO( **111 million** parameters) can be summarized as follows:

1. The model takes a text instruction and a set of images as inputs
2. The object descriptions are extracted from the language instruction, and together with the initial image, fed into a frozen Vision-Language Model (**VLM**) (**Owl-ViT**) to output 2D object coordinates in the image
3. The Owl-Vit has a backbone **ViT-B/ViT-H/ViT-L** with size **86/307/632** million respectively. The paper does mention which one they use.
4. The text instruction, excluding the object descriptions, is encoded in an embedding space of a Language Model (LLM).
5. The images ( the segmentation mask is concatenated channel-wise to the current image together with the recent image ) are processed by an EfficientNet conditioned on the text embedding via Feature-wise Linear Modulation (FiLM) layers
6. A Token Learner is used to compute a small set of tokens, followed by a Transformer to attend over these tokens and produce discretized action tokens
7. The resulting action tokens are used to predict actions for the robot, taking into account the object localization information and the instruction
