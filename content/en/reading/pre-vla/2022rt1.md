# RT-1: Robotics Transformer for Real-World Control at Scale

**Authors**: Anthony Brohan et al.
**Organization**: Google Robotics
**Year**: 2022
**Paper**: [https://robotics-transformer1.github.io/assets/rt1.pdf](https://robotics-transformer1.github.io/assets/rt1.pdf)

---

# Problem

Traditional robot policies are typically trained for **single tasks** using relatively small datasets.

Common approaches include:

- behavioral cloning
- reinforcement learning
- task‑specific policy networks

These methods often suffer from several limitations:

- policies are trained for **narrow tasks**
- datasets are **small and fragmented**
- models generalize poorly to **new objects or environments**

At the same time, other fields such as NLP and computer vision have shown that **scaling models and datasets** can dramatically improve performance.

This raises an important question:

> Can robot policies also benefit from **large-scale multi-task training**?

---

# Why it matters

If robot learning can scale similarly to language and vision models, we could potentially train:

- **generalist robot policies**
- capable of performing many tasks
- trained from large shared datasets

Such systems could:

- reuse knowledge across tasks
- generalize to new objects
- reduce the need for task‑specific engineering

However, several challenges arise:

- robot observations are **high-dimensional images**
- actions must be predicted **in real time**
- robot datasets are much smaller than web-scale datasets

A practical architecture must therefore balance:

- expressiveness
- computational efficiency
- real-time control requirements

---

# Attempt

RT-1 proposes to model robot policies using a **Transformer-based architecture**.

The core idea is to treat robot control as a **sequence modeling problem**.

The model receives:

- a short history of images
- a natural language instruction

and predicts **discretized robot actions**.

Pipeline:

image observations + instruction
↓
vision-language encoding
↓
Transformer policy
↓
action tokens

By framing robot control as token prediction, RT‑1 can leverage techniques from large-scale sequence models.

---

# Model

## Vision-Language Encoding

RT‑1 uses a **FiLM-conditioned EfficientNet-B3** to process images.

Key details:

- input resolution: 300 × 300
- output feature map: 9 × 9 × 512
- flattened into **81 visual tokens**

Language instructions are embedded using the **Universal Sentence Encoder (USE)**.

The language embedding conditions the visual encoder via **FiLM layers**, allowing the visual features to depend on the instruction.

---

## Token Compression

Processing 81 visual tokens directly in a Transformer would be computationally expensive.

RT‑1 introduces a **TokenLearner module** to reduce the sequence length.

The module:

- attends over the 81 tokens
- selects the most informative features
- compresses them into **8 tokens**

This significantly reduces computation while preserving relevant information.

---

## Action Decoder

The compressed tokens are fed into a **decoder-only Transformer**.

Key properties:

- 8 self-attention layers
- about **19M parameters**

The full RT‑1 system contains roughly **35M parameters** and can run at **3 Hz** in real-world control loops.

---

## Action Representation

Robot actions are **discretized into tokens**.

Action dimensions include:

Arm control (7 dimensions):

- x, y, z
- roll, pitch, yaw
- gripper opening

Base control (3 dimensions):

- x
- y
- yaw

Mode variable:

- arm control
- base control
- episode termination

Each dimension is discretized into **256 bins**, converting continuous control into a token prediction task.

---

# Failure / Limitation

Despite its strong performance, RT‑1 has several limitations.

### Discretized action space

Discretization simplifies training but may reduce control precision compared with continuous policies.

---

### Behavior cloning formulation

RT‑1 primarily relies on imitation learning.

This means the policy may struggle with:

- recovery from errors
- exploration
- long-horizon optimization

---

### Limited control frequency

RT‑1 operates at **3 Hz**, which is sufficient for many manipulation tasks but too slow for high-frequency control.

---

### Embodiment specificity

Although tasks are diverse, the model is still trained on a particular robot setup.

Generalizing across different robot embodiments remains challenging.

---

# Results

RT‑1 is trained on a large real-world robot dataset:

- ~130k demonstrations
- collected over **17 months**
- from **13 robots**

The dataset includes tasks such as:

- pick
- place
- open / close drawers
- retrieving objects from drawers

Evaluation shows improvements in:

- multi-task performance
- robustness to distractors
- generalization to new environments

---

# Key Idea

The main insight of RT‑1 is that **robot policies can be trained as large-scale sequence models**.

By representing actions as tokens and combining vision and language inputs, the system can learn a multi-task robot policy from diverse data.

---

# Takeaway

RT‑1 is one of the earliest demonstrations that **scaling approaches from NLP and vision may also apply to robotics**.

Conceptually, it introduces a new paradigm:

> robot control as sequence modeling over vision, language, and action tokens.

This idea later influenced the development of **Vision-Language-Action (VLA)** models such as RT‑2 and OpenVLA.
