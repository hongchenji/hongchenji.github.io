**RT-H**

_by Google DeepMind Robotics & Stanford_

While language can help encode shared structure between tasks (“grasp apple”, “grasp can”), for diverse, semantically different tasks with diverse language descriptions it gets harder for models to identify shared similarities (e.g. when pushing a button or picking up a can, moving forwards would be relevant for both) and thus improve data sharing & generalization. Additionally, this allows for easier user corrections of the policy using language).

For this RT-H introduces a hierarchy (task → language motion → low-level actions) via an intermediate prediction layer.

**Architecture / Method**

- same PaLI-X 55B architecture as RT-2 + ViT encoder
- action tokens are discretized in 245 bins
- uses a single VLM but 2 queries:
  - language motion query: encoder sees language task g and image observation o & decoder predicts language motions z (ex: output language motion is “moves arm forward and close gripper”)
  - action query: encoder sees g, o and z & decoder predicts robot action a (output action is delta positions/rotations of EE + gripper open/close)
  - full action policy is composition of the two learned policies from these queries
  - future work: consider multiple steps of abstraction in intermediate levels such as splitting in subtasks
- uses automated labeling scheme based on proprioception (as VLMs do not have strong spatial prior for robot actions and human annotation is prone to inaccuracy and misjudgement): text description of dominant spatial movements (threshold based)
- Data: 100k demonstrations (dataset from RT-1/RT-2 plus new dataset
- User correction: User can intervene with keyboard press to start correction mode and give language motion correction which is passed to action query, then at fixed frequency user can a) send new correction b) keep running previous correction or c) exist correction mode
  - learning from correction: only learning small changes language motion necessary, thus smaller language motion space (instead of large action space necessary) → since there is only one model and we cannot just train this part, co-training with original dataset ensures action prediction performance
