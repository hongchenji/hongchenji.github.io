# FAST：用于 Vision-Language-Action 模型的高效动作 Tokenization

**作者：** Karl Pertsch, Kyle Stachowicz, Brian Ichter, Danny Driess, Suraj Nair, Quan Vuong, Oier Mees, Chelsea Finn, Sergey Levine
**机构：** Physical Intelligence, UC Berkeley, Stanford
**年份：** 2025

论文：
[https://arxiv.org/pdf/2501.09747](https://arxiv.org/pdf/2501.09747)

---

# Problem

在 Vision-Language-Action（VLA）模型中，机器人动作通常通过 **binning discretization（分桶离散化）** 进行 tokenization。

这种方法通常：

- 按 **每个动作维度** 离散化
- 按 **每个时间步** 生成一个 token

虽然实现简单，但在 **高频机器人控制数据** 中会带来明显问题。

机器人控制信号通常是 **连续且平滑的轨迹**，这意味着：

- 相邻时间步的动作高度相关
- token 序列包含大量重复信息

因此会产生两个核心问题：

- token 序列过长
- 每个 token 的信息量很低

---

# Why it matters

在 VLA 模型中，**动作 token 序列长度直接影响训练效率**。

如果动作被简单离散化：

- Transformer 需要处理非常长的序列
- 上下文窗口被低信息 token 占用
- 模型很容易学会“复制上一个 token”

结果是：

- 学习信号变弱
- 训练效率下降
- 模型难以扩展

因此关键问题是：

> 是否可以在保持动作信息的同时，大幅减少 token 数量？

---

# Attempt

FAST 提出一种 **基于压缩的动作 tokenization 方法**。

核心思想是：

不要直接在时间域对动作离散化，而是先 **压缩动作信号**。

FAST pipeline 包含三个主要步骤：

1. 动作归一化
2. 频域变换（DCT）
3. 使用 BPE 进行 token 压缩

这样可以将一个长动作序列转换为 **更短但信息量更高的 token 序列**。

---

# Method

## 1. Action Normalization

在 tokenization 之前，动作需要归一化。

每个动作维度使用数据集的：

- **1% 分位数**
- **99% 分位数**

映射到区间：

[-1,1]

这样可以避免极端值对表示造成影响。

---

## 2. Discrete Cosine Transform (DCT)

FAST 对每个动作维度独立应用 **DCT**。

DCT 将动作序列从：

时间域 → 频率域

一个轨迹可以表示为多个不同频率余弦函数的叠加。

关键观察是：

- **低频成分包含主要运动结构**
- **高频成分通常是冗余或噪声**

因此可以通过丢弃较小的高频系数来实现压缩。

---

## 3. Matrix Construction

对所有动作维度应用 DCT 后：

- 得到一个频率系数矩阵

该矩阵随后：

- 按 **列优先（column-wise）** 展平成一维序列

该序列即为压缩后的动作表示。

---

## 4. Byte Pair Encoding (BPE)

接下来使用 **Byte Pair Encoding（BPE）** 对序列进一步压缩。

BPE 会学习：

- 数据中频繁出现的系数组合

并将其合并为新的 token。

最终得到：

**固定大小词表的动作 token**。

需要注意：

- FAST 中唯一需要学习的部分是 **BPE 词表**
- 通常针对 **每个数据集单独训练 tokenizer**

---

# FAST+

论文还提出 **FAST+**，即通用机器人 tokenizer。

FAST+ 在大约 **100 万条机器人动作数据** 上训练。

因此可以跨：

- 不同数据集
- 不同机器人平台

直接使用。

这说明未来可能存在：

**统一的机器人动作 tokenizer。**

---

# Results

## Training Speed

FAST 可以显著加快训练速度。

论文报告示例：

π0 vs π0-FAST

训练速度提升约：

**5×**。

主要原因是 FAST 产生的 token 序列更短。

---

## Token Compression

FAST 可以显著减少每个动作片段的 token 数量。

特别是在 **高频机器人控制数据** 中。

同时 BPE 可以避免大量：

重复的 0 token

这些 token 在传统 discretization 中非常常见，并会降低训练效率。

---

## Inference Speed

一个有趣的现象是：

FAST 在推理阶段 **反而更慢**。

论文报告示例：

π0：约 100 ms
π0-FAST：约 750 ms

原因包括：

- 需要更多自回归 decoding
- 使用大型语言模型 backbone
- 而不是 π0 中较小的 action expert

---

# Failure / Limitation

FAST 也存在一些权衡。

### 推理延迟增加

压缩提升了训练效率，但可能增加推理时间。

### 数据集依赖

FAST tokenizer 通常针对特定数据集训练。

### 额外 preprocessing

方法需要额外步骤：

- DCT
- BPE 训练

---

# Key Idea

FAST 的核心思想是：

**机器人动作序列本质上是高度冗余的信号。**

因此动作 tokenization 更像是一个 **信号压缩问题**，而不是简单的离散化问题。

---

# Takeaway

FAST 的整体 pipeline 可以总结为：

trajectory
→ normalization
→ DCT (frequency transform)
→ flatten
→ BPE compression
→ action tokens

通过在 tokenization 前对动作信号进行压缩，FAST 能够：

- 减少 token 序列长度
- 提升训练效率
- 改善 VLA 模型的可扩展性。
