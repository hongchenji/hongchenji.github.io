# π0.5：层级式 Vision-Language-Action Flow 模型

**机构**：Physical Intelligence
**年份**：2025

---

# Problem

早期的 Vision-Language-Action（VLA）模型（例如 RT‑1、RT‑2 和 π0）证明：

- 机器人策略可以通过 Transformer 架构学习
- 多模态输入（视觉 + 语言）可以直接映射到动作
- 大规模数据可以显著提升泛化能力

然而这些系统在 **长时序任务（long‑horizon tasks）** 上仍然存在明显困难。

典型问题包括：

- 任务需要很多连续步骤
- 家庭环境复杂且变化多
- 在新场景中泛化能力有限
- 难以保持长期一致的任务规划

原因在于：

这些模型通常直接学习

observation → action

而没有显式建模 **任务结构**。

因此一个关键问题是：

> 是否可以将 **高层规划** 与 **低层连续控制** 结合，从而提升长时序机器人任务能力？

---

# Why it matters

现实中的机器人任务通常具有明显的层级结构。

例如：

做咖啡

可能包含以下子步骤：

1. 找到杯子
2. 将杯子放在咖啡机下
3. 按下按钮
4. 等待咖啡完成

如果策略没有显式的子任务结构，模型必须隐式学习整条动作链。

这会导致：

- 训练难度增大
- 长时序任务不稳定
- 泛化能力下降

如果模型能够预测 **高层子任务（subtask）**：

- 长时序规划会更稳定
- 新场景泛化能力更好
- 任务结构更加清晰

---

# Attempt

π0.5 在 π0 的基础上引入 **层级式结构（hierarchical architecture）**。

核心思想是：

将机器人行为分为两个层次：

1️⃣ **高层子任务预测（language‑like planning）**
2️⃣ **低层连续动作生成（flow‑based control）**

同时引入：

- FAST action tokenization（用于预训练）
- 两阶段训练流程
- 多模态 token routing

因此 π0.5 同时结合了：

- 语言生成
- 动作 token 预训练
- 连续动作 flow matching
- 层级式任务规划

---

# Model

## 多模态输入 Token

模型处理一系列 **多模态 token**。

每个 token (x_i) 可以表示：

- 文本 token
- 图像 patch
- 机器人状态
- flow matching 中的噪声动作

具体包括：

- 文本 token (x_i^w)
- 图像 patch (x_i^I)
- 动作值 (x_i^a)

观测 (o_t) 和语言指令 (l) 构成 token 序列的前缀。

---

## Token 类型路由

函数 (\rho(x_i)) 表示 token 类型。

它决定：

- token 进入哪个 encoder
- Transformer 中使用哪组 expert 权重

例如：

- text token → embedding matrix
- image token → vision encoder
- action token → robotics action expert

这种设计可以复用视觉语言模型能力，同时加入机器人专用模块。

---

## 两阶段输出结构

π0.5 同时预测 **语言计划** 与 **连续动作**。

输出被划分为两部分：

- 前 (M) 个输出：文本 token logits
- 后 (H) 个输出：动作输出

文本 token 可以被采样为 **高层子任务计划**。

这些计划随后用于条件化生成连续机器人动作。

---

## Flow Matching 连续动作生成

与 π0 一样，π0.5 使用 **conditional flow matching** 生成连续动作。

推理阶段需要执行：

- 大约 10 次去噪步骤

从而得到平滑的动作轨迹。

---

# FAST Action Tokenization

π0.5 引入 **FAST** 来加速预训练。

目标是将连续动作轨迹转换为离散 token。

流程如下：

1. 将动作维度归一化到 [-1,1]
2. 对每个维度应用 **DCT（离散余弦变换）**
3. 将结果展平
4. 使用 **BPE（Byte Pair Encoding）** 压缩表示

这样可以得到可用于自回归预测的动作 token。

FAST 可以让模型在预训练阶段更早学习到动作结构。

---

# Training

π0.5 使用 **两阶段训练流程**。

## Stage 1：Pretraining

预训练阶段模型表现为一个标准 VLM。

模型预测：

- 文本 token
- 物体位置
- FAST 动作 token

训练长度：

- 约 280k gradient steps

监督信号包括：

- 子任务标签
- 物体定位
- 图像描述
- 问答任务

---

## Stage 2：Post‑training

在后训练阶段：

- 启用 action expert
- 激活 flow matching loss
- 引入额外 expert 权重

训练继续约：

- 80k gradient steps

---

# Loss

训练目标为组合损失：

[
L = H + \alpha F
]

其中：

- H：token 预测 cross‑entropy
- F：flow matching loss

因此模型同时学习：

- 语言规划
- 连续动作控制

---

# Failure / Limitation

尽管 π0.5 引入了层级结构，但仍然存在挑战。

### 数据需求

层级模型通常需要：

- 子任务标注
- 语言标签

这些标注成本较高。

### 训练复杂度

系统同时包含：

- token 预测
- flow matching
- hierarchical planning

训练流程较复杂。

### 数据配方敏感

模型性能高度依赖：

- 数据混合方式
- 训练 recipe

---

# Key Idea

π0.5 的核心思想是：

**机器人策略应该具有层级结构。**

系统首先预测高层子任务，然后在该子任务条件下生成连续动作。

---

# Takeaway

π0.5 将 VLA 模型推进到一个更结构化的阶段：

perception → high‑level plan → continuous control

通过结合层级规划与 flow‑based action generation，π0.5 试图提升机器人在真实环境中的长时序任务能力。
